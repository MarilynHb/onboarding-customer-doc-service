The document Api, with the genuinely new part: handling file uploads and downloads over HTTP without loading whole files into memory. Same controller-and-DI pattern you learned in Part 1, plus streaming.

What you'll build (sketch)

  • A DocumentsController with upload (POST with IFormFile), list, get metadata, download (FileStreamResult), submit, approve, reject.
  • The same fake auth middleware as Part 1, distinguishing customer vs admin and the document owner.
  • A .http file, plus a manual test: upload a large file and confirm memory doesn't balloon.

What's new compared to Part 1

  • Multipart file uploads with IFormFile.
  • Streaming downloads, no buffering the whole file in memory.
  • The Api now containerized alongside Azurite, the database, and the customer service.
