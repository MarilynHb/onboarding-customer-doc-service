This is an outline. We'll flesh it out together after Part 1.

Same as Part 1's contracts step, but with one important addition: alongside IDocumentStore (the metadata database), you also define IDocumentStorage (the file storage). Two separate concerns, two separate interfaces, both defined here, both implemented later in the Store layer.

What you'll build (sketch)

  • IDocumentStore for the metadata (same shape as ICustomerStore).
  • IDocumentStorage for the file: UploadAsync(stream), DownloadAsync(reference), DeleteAsync(reference).
  • DTOs and validators, as before.

What's new compared to Part 1

  • The split between "where the metadata lives" and "where the file lives" is the key new idea. The service will use both.
  • Streams enter the picture: the storage interface deals in Stream, not byte[]. (We'll cover why in Step 6.)
