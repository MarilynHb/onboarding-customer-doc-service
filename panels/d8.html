What you'll build (sketch)

  • One polished docker compose up that brings up the whole system: customer service, document service, MSSQL (two databases), Azurite, and the Blazor UI.
  • The UI extended so a customer can upload a document during onboarding and an admin can review it, exercising the full cross-service flow end to end.
  • A top-level README for the whole system.

Stretch goals (pick what interests you)

  • Tests: real coverage on both services, plus integration tests with WebApplicationFactory.
  • SAS URLs: let the browser download blobs directly via short-lived signed URLs instead of streaming through the Api.
  • Audit log: record every status transition; let admins view the history.
  • Structured logging: add Serilog across both services.
  • Resilience: add Polly retry policies on the cross-service HTTP calls.
  • Document expiry: identity documents expire after 12 months, with an automatic transition.
