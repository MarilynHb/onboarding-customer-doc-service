This is an outline. We'll flesh it out together after Part 1.

The document service's biggest new concept: storing actual files in Azure Blob, emulated locally with Azurite. You build the DocumentService class against both interfaces, and implement IDocumentStorage against Azurite.

What you'll build (sketch)

  • DocumentService with use cases: upload, list mine, get by id, resubmit, approve, reject.
  • Azurite running in Docker (one more container).
  • An AzureBlobDocumentStorage implementing IDocumentStorage, using the Azure.Storage.Blobs SDK and the magic UseDevelopmentStorage=true connection string.
  • The upload use case: store the file via IDocumentStorage, save the metadata + storage reference via IDocumentStore.

What's new compared to Part 1

  • Two backing stores instead of one: the metadata database and the blob storage.
  • Azurite as a local emulator for Azure Blob (no Azure account or cost).
  • The Microsoft Storage Explorer for poking at stored blobs by hand.
