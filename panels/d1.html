What you'll build

  • A new DocumentService solution with the same six-project layout.
  • A short note in notes/ on what's different this time: there's a file to store somewhere (not just rows in a table), and eventually another service will call this one.

What's new compared to Part 1

  • Documents have an actual file, not just metadata. That file needs somewhere to live (blob storage, coming in Step 4).
  • The state machine is richer: Draft, Submitted, UnderReview, Approved, Rejected, with resubmission.
  • This service will eventually be called by the customer service, not by a user directly.
