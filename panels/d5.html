This is an outline. We'll flesh it out together after Part 1.

This step is mostly a repeat of Part 1's Store step, you've done EF Core and MSSQL already. The document service gets its own database (separate from the customer one) for document metadata. The blob side is already handled from Step 4.

What you'll build (sketch)

  • DocumentsDbContext and an EfDocumentStore implementing IDocumentStore.
  • A second database in the same MSSQL container (separate database, not a separate server).
  • Migrations for the document metadata schema.

What's new compared to Part 1

  • Two databases now exist (CustomerDb and DocumentsDb), reinforcing "each service owns its own data."
  • Very little new EF Core to learn, this is consolidation.
