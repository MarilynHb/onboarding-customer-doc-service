Documents Service · Milestone 5

Web layer and fake auth

~5 days First end-to-end File handling

The first end-to-end vertical slice. By the end of this milestone, you have a working REST API for documents that you can hit with Postman or .http files. Customers (with the right role) can upload, list, and download their own documents. Admins can list everything.

What you'll build

The Documents.Api project:

  • Controllers:
    • POST /api/documents (upload)
    • GET /api/documents (list current user's)
    • GET /api/documents/{id} (single document)
    • GET /api/documents/{id}/content (download stream)
    • POST /api/documents/{id}/submit
    • POST /api/documents/{id}/approve (admin)
    • POST /api/documents/{id}/reject (admin)
  • A fake auth middleware that reads X-User-Id and X-User-Role headers and attaches them as claims. Use [Authorize(Roles = "Admin")] on admin endpoints.
  • File upload via IFormFile. File download via FileStreamResult.
  • A .http file with sample requests.
  • Updated docker-compose.yml that runs Documents.Api in a container.

What you'll learn

  • ASP.NET Core controllers, routing, model binding, action results
  • Middleware and the request pipeline
  • File upload patterns (IFormFile, multipart form data)
  • Streaming file downloads without buffering in memory
  • Claims-based authorization
  • Dockerizing a .NET application
  • Service-to-container networking in Docker Compose

Suggested approach

  1. Build the controllers first, without auth. Make sure CRUD endpoints work end-to-end with your .http file.
  2. Add the fake auth middleware. Place it before UseAuthorization(). Test admin-only endpoints reject non-admin headers.
  3. Handle file uploads with IFormFile. Don't read the whole file into memory. Test: can you upload a 100MB file without your app's memory ballooning?
  4. Handle downloads with FileStreamResult and let ASP.NET Core stream it.
  5. Add a Dockerfile for Documents.Api. Use multi-stage build. Add it as a service in docker-compose.yml. Inside Docker, Azurite is at azurite, not localhost.
  6. Run docker compose up and confirm everything works in containers.

C# depth moment: delegates and the middleware pipeline. A middleware in ASP.NET Core is literally a function that takes a RequestDelegate and returns one. This is Func<> and Action<> made concrete. Write your auth middleware twice: once with the app.Use(async (context, next) => ...) lambda syntax, once as a full class. Compare them.
