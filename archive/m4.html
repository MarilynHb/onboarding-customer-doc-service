Documents Service · Milestone 4

Infrastructure: MSSQL and Azurite

~6 days EF Core Docker enters

Two infrastructure concerns to wire up: the database for metadata, and Azure Blob (via Azurite) for files. This is also where Docker enters the project, because we'll run both MSSQL and Azurite as containers from now on.

What you'll build

  • A docker-compose.yml at the root, starting two containers: MSSQL (or azure-sql-edge if you're on an Apple Silicon Mac) and Azurite
  • EF Core wired up: DocumentsDbContext, entity configurations, an initial migration
  • A real DocumentRepository backed by EF Core
  • An IDocumentStorage interface in Documents.Application
  • An AzureBlobDocumentStorage implementation in Documents.Infrastructure
  • All connection strings via appsettings.json and environment variables

By the end, you should be able to swap the in-memory repository for the real one in Program.cs, and nothing else changes. That's the lesson.

What you'll learn

  • EF Core: DbContext, configurations, migrations, async queries, change tracking
  • Azure Blob via Azurite: containers, blobs, connection strings, the SDK
  • Docker basics: images, containers, ports, volumes
  • Microsoft Storage Explorer for inspecting Azurite contents
  • Why we use streams for file upload/download instead of byte arrays
  • Configuration in .NET: appsettings, environment variables, the layered system

Suggested approach

  1. Start with Docker. Get Azurite running first, in isolation:
services:
  azurite:
    image: mcr.microsoft.com/azure-storage/azurite
    ports:
      - "10000:10000"
      - "10001:10001"
      - "10002:10002"
    command: "azurite --blobHost 0.0.0.0 --queueHost 0.0.0.0 --tableHost 0.0.0.0 --loose --skipApiVersionCheck"

Run docker compose up azurite. Install Microsoft Azure Storage Explorer and connect to it (Attach to a local emulator). Create a container and upload a blob manually before you write any code.

  1. Add MSSQL to docker-compose. Intel/Windows/Linux: mcr.microsoft.com/mssql/server. Apple Silicon: mcr.microsoft.com/azure-sql-edge. Verify you can connect from Azure Data Studio.
  2. Now wire up EF Core. Create DocumentsDbContext, register it in Program.cs, write your first migration:
dotnet ef migrations add InitialCreate \
  --project src/Documents.Infrastructure \
  --startup-project src/Documents.Api

dotnet ef database update \
  --project src/Documents.Infrastructure \
  --startup-project src/Documents.Api
  1. Replace your in-memory IDocumentRepository with a real EF Core-backed one. Your handlers should still work, with the only change being the DI registration.
  2. Define IDocumentStorage in Documents.Application:
public interface IDocumentStorage
{
    Task<string> UploadAsync(Stream content, string fileName, string contentType, CancellationToken ct);
    Task<Stream> DownloadAsync(string storageReference, CancellationToken ct);
    Task DeleteAsync(string storageReference, CancellationToken ct);
}
  1. Implement AzureBlobDocumentStorage using the Azure.Storage.Blobs NuGet package. Connection string for Azurite is UseDevelopmentStorage=true.
  2. Update UploadDocument handler to call IDocumentStorage and save the storage reference to the database.
