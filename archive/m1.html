Documents Service · Milestone 1

Orientation and foundations

~3 days No code yet Reading + reflection

You won't write any project code yet. The work here is reading, exploring, and writing a short reflection. It feels slow but it's the highest-leverage time in the whole project.

What you'll build

A written reflection document. Two parts:

  1. Your predictions of what each project in the Jason Taylor Clean Architecture template does, before you generate it locally.
  2. Your comparison of those predictions to what you actually find when you generate it.

Markdown file, kept in your project repo as notes/orientation.md.

What you'll learn

  • The actual difference between C# (the language) and .NET (the platform). This is the single most common confusion for new .NET developers.
  • What a .sln file is, what a .csproj file is, what bin and obj are, what Program.cs does.
  • What Clean Architecture is, at a conceptual level, before you see any code.

Suggested approach

  1. Read the resources below in order. Take notes as you go.
  2. Open the Jason Taylor template on GitHub (don't clone it yet). Look at the folder structure. Read the README. For each project (Domain, Application, Infrastructure, Web), write 1-2 sentences predicting what's inside and why.
  3. Now generate the template locally:
dotnet new install Clean.Architecture.Solution.Template
mkdir DocumentSystem && cd DocumentSystem
dotnet new ca-sln --output .
  1. Explore. Open every .csproj. Open Program.cs. Look at the dependencies between projects. Compare what you actually find to your predictions. Write the comparison.

Questions to be able to answer

  1. What's the difference between C# 12 and .NET 8? Are they the same thing?
  2. Why does the template have a separate Domain project? What would break if Domain referenced Infrastructure?
  3. What's Program.cs actually doing in modern .NET?
  4. Where in the template would you put a new "business rule"? Where would you put a database query? Why?
